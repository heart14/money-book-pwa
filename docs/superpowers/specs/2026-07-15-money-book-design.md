# 钱书 V2 — 产品设计说明书

## 1. 产品概述

| 项目 | 内容 |
|------|------|
| 产品名称 | 钱书 |
| 定位 | 本地优先、零后端依赖的个人记账 PWA |
| 核心理念 | 记账简单快速 · 分类清晰全面 · 财务逻辑严谨 · 数据安全可靠 |
| 目标用户 | 个人日常记账，日均 1-5 笔流水 |
| 适配平台 | 移动端 (≤768px) + 桌面端 (>768px) |

## 2. 技术架构

### 2.1 技术栈

| 模块 | 选型 | 职责 |
|------|------|------|
| 框架 | Vue 3 + TypeScript (Composition API) | 响应式 UI 与业务逻辑组合 |
| 样式 | UnoCSS | 原子化 CSS，iOS 设计 Token |
| 数据库 | Dexie.js (IndexedDB) | 本地持久化，liveQuery 响应式查询 |
| 状态管理 | Pinia | 缓存计算结果、UI 状态 |
| 图表 | ECharts (按需导入) | 统计页图表渲染 |
| PWA | vite-plugin-pwa | Service Worker + manifest + 离线缓存 |
| 工具库 | @vueuse/core | useObservable 桥接 Dexie ↔ Vue 响应式 |
| 图标 | Lucide Icons 或 Phosphor Icons | 统一 SVG 线性图标，2px 描边，圆头端点 |

### 2.2 架构分层

```
PWA Shell (Manifest + SW + navigator.storage.persist())
│
Vue 3 + TypeScript
├── Pages:  记账 / 明细 / 账户 / 统计 / 设置
├── Composables: useBookkeeping, useTransactions, useAccounts, ...
├── Pinia Stores: accountStore, categoryStore, transactionStore, uiStore
├── Dexie.js (liveQuery → useObservable → computed → UI)
│
UnoCSS (iOS Design Tokens)
└── SF Pro 风格字体 · 毛玻璃效果 · 灵动圆角 · 柔和阴影
```

### 2.3 响应式适配策略

- **移动端 (≤768px)**：底部 Tab 导航，全屏键盘录入，单列布局
- **桌面端 (>768px)**：左侧侧边栏导航，记账改为弹窗/浮层模式，统计页多列看板

## 4. 数据模型设计

### 4.1 数据库表结构 (Dexie.js)

#### accounts（账户表）

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | `++number` | 自增主键 |
| `name` | `string` | 账户名称 |
| `groupId` | `'liquid' \| 'restricted' \| 'claim' \| 'debt'` | 账户分组 |
| `balance` | `number` | 当前余额（负债存负数） |
| `icon` | `string` | Emoji 图标 |
| `sort` | `number` | 排序序数 |

#### categories（分类表）

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | `++number` | 自增主键 |
| `type` | `'expense' \| 'income'` | 支出或收入 |
| `parentId` | `number \| null` | 一级为 null，二级指向一级 id |
| `name` | `string` | 分类名称 |
| `icon` | `string` | Emoji 图标 |
| `sort` | `number` | 排序序数 |

#### transactions（流水表）

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | `++number` | 自增主键 |
| `type` | `'expense' \| 'income' \| 'transfer'` | 流水类型 |
| `amount` | `number` | 金额（正数） |
| `fromAccountId` | `number \| null` | 转出/支付账户 |
| `toAccountId` | `number \| null` | 收入/转入账户 |
| `categoryId` | `number \| null` | 分类（转账时为 null） |
| `tags` | `string[]` | 标签数组 |
| `note` | `string` | 备注 |
| `date` | `string` | 日期 YYYY-MM-DD |
| `time` | `string` | 时间 HH:mm |

#### recurring_rules（周期记账规则表）

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | `++number` | 自增主键 |
| `type` | `'expense' \| 'income' \| 'transfer'` | 类型 |
| `amount` | `number` | 金额 |
| `fromAccountId` | `number` | 转出/支付账户 |
| `toAccountId` | `number \| null` | 转入账户 |
| `categoryId` | `number \| null` | 分类 |
| `tags` | `string[]` | 标签 |
| `note` | `string` | 备注 |
| `dayOfMonth` | `number` | 每月第几日 (1-28) |
| `enabled` | `boolean` | 是否启用 |
| `lastExecuted` | `string \| null` | 上次执行日期 |

### 4.2 数据一致性规则

- **原子更新**：任何流水变动（增/删/改）通过 `db.transaction` 同时更新 `accounts.balance`
- **转账隔离**：`type = 'transfer'` 时 `categoryId = null`，系统在交互层面拦截将其记为收支的操作
- **负债余额**：负债账户余额存负数，计算净资产时 `SUM(balance)`

### 4.3 预设数据字典

采用两级分类体系（一级 8 个，二级细分）+ 标签系统。

#### 预设账户组与账户

| 账户组 | 预设账户 | 类型标识 |
|--------|----------|----------|
| 流动性资产 | 微信零钱、支付宝余额、银行卡、支付宝小荷包 | `liquid` |
| 限制性资产 | 公积金账户、投资理财账户 | `restricted` |
| 债权 | 借出款项 | `claim` |
| 负债 | 信用卡、借入款项、房贷本金 | `debt` |

#### 预设支出分类（8 个一级）

- 🏠 居住：房租房贷、生活缴费、家居家装
- 🍽️ 餐饮：餐饮美食、生鲜食材、休闲食品
- 🚗 交通：公共交通、汽车消费、停车通行、汽车加油、汽车充电
- 🛍️ 购物：日用百货、服饰装扮、美妆护理、数码家电
- 🎭 休闲：虚拟消费、文娱休闲、兴趣潮玩、运动健身、旅游出行
- 👨‍👩‍👧 家庭：文化教育、医疗保健、宠物消费
- 🤝 人情：红包转账、节日礼物
- 🧮 特别：订婚、婚礼、购车、购房、其它

#### 预设收入分类（3 个一级）

- 💼 主动收入：税后薪酬、绩效奖金、兼职外包、公积金入账
- 💰 被动收入：利息分红、资本利得
- 🎁 其它收入：转卖返现、红包转账

## 5. 页面详细设计

### 5.1 记账页（默认首页）

**工作流：三步记账**
1. 输入金额（数字键盘）
2. 选择分类（两级宫格：一级横向滚动，点击展开二级）
3. 确认（自动完成，支持快捷记忆上次账户和分类）

**界面布局：**
- 顶部：收入/支出/转账 三段式切换器
- 金额区：大字 `¥0.00` 显示
- 账户选择器：毛玻璃卡片，点击切换
- 一级分类：4 列宫格，Emoji + 名称
- 二级分类：Chip 标签横向滚动，选中高亮蓝底
- 备注与标签：可折叠展开
- 底部：iOS 计算器风格数字键盘

**转账模式：** 隐藏分类区域，显示"从 → 到"双账户选择器。

### 5.2 明细页

**界面布局：**
- 顶部操作栏：日期选择 + 搜索按钮
- 分类筛选 Chip：横向滚动，点击切换
- 流水列表：按日期分组（今天/昨天/本周/更早），每组显示日汇总
- 每条记录：Emoji 图标 + 分类名 + 账户 + 时间 + 标签 + 金额（支出红色、收入绿色）
- 点击记录 → 底部弹窗查看/编辑/删除

### 5.3 账户页

**界面布局：**
- 顶部：深色净资产卡片（大字余额 + 总资产/总负债拆解）
- 账户分组列表（可折叠展开）：
  - 流动性资产（余额为正，黑色）
  - 限制性资产（余额为正，黑色）
  - 债权（余额为正，黑色）
  - 负债（余额为负，红色显示）
- 每组显示组合计
- 支持新增账户、编辑、排序
- 点击账户 → 进入该账户的流水列表

### 5.4 统计页

**功能点：**
- 时间维度切换：月/年/自定义
- 收入/支出概览卡片（绿色/红色）
- 月度趋势柱状图（收入绿柱 + 支出浅红柱）
- 支出构成环形图（含图例和百分比）
- 支出排行（1-3 名带序号、金额、占比）
- 标签聚合（所有标签汇总展示，点击跳转筛选）

**桌面端看板模式：** 图表并排多列展示。

### 5.5 设置页

**功能分组：**
- **安全**：PIN 码锁（开启/关闭/修改）
- **数据管理**：导出备份（JSON 文件下载）、导入数据、彻底销毁
- **分类与标签**：管理分类（显示/隐藏二级分类）、管理标签（新增/删除）
- **周期记账**：管理周期规则（新增/编辑/启用/停用）
- **关于**：版本号

## 6. 预设分类与种子数据

- 应用首次启动时初始化预设账户和分类数据
- 检查 IndexedDB 是否为空，为空则写入种子数据
- 分类采用原钱书 V1 PRD 的分类体系（不含拿铁因子等洞察卡片相关字段）

## 7. PWA 与离线策略

- manifest.json：`display: standalone`，配置应用名"钱书"与图标
- Service Worker：vite-plugin-pwa 自动生成，静态资源强缓存
- 持久化：首次加载调用 `navigator.storage.persist()` 申请免清理保护
- 安全：PIN 码锁 + 数据加密导出 + 一键销毁

## 8. 非功能需求

- 记账操作不超过 3 步完成
- 所有页面在移动端和桌面端均正常运行
- 数据不依赖任何后端服务，完全本地存储
- 无需用户注册/登录
