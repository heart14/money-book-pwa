# 《钱书》V1.0 产品需求文档 (PRD)

## 1. 产品概述

*   **产品名称**：钱书
*   **产品定位**：防微杜渐的财务逻辑、真实全览的资产负债、极简克制的数据录入。
*   **核心架构**：本地优先，零后端依赖，纯前端 PWA。
*   **技术栈**：Vue 3 + Vite + Vant 4 + Dexie.js + ECharts + vite-plugin-pwa。

## 2. 核心产品哲学与业务红线（开发必读）

1.  **按“财务实质与生活领域”分类，而非“物理形态”**：
    *   *越野跑鞋*归“运动健身”而非“衣鞋配饰”；*专业软件订阅*归“职场办公”而非“数码软硬”；*盲盒手办*归“兴趣潮玩”而非“日用百货”。
    *   分类是为了看清钱去哪了（目的），而不是清点家里买了什么（形态）。
2.  **流转隔离法则（最高红线）**：强制隔离“收支”与“转账”。还信用卡、买理财、存小荷包、借还钱，**一律只能记“转账”**，系统必须在交互层面拦截将其记为支出的操作。
3.  **刷信用卡记支出**：使用信用卡/花呗消费时，记“支出”，账户选对应负债账户，体现真实负债增加。
5.  **公积金双记录**：公积金缴存记“收入-公积金入账”，公积金抵扣房贷记“支出-房贷”，真实反映居住成本与资产全貌。

## 3. 功能模块详细需求

### 3.1 极速记账页
*   **默认视图**：底部数字键盘，顶部滑动切换 <支出 / 收入 / 转账>。
*   **智能防错引擎（强制逻辑）**：
    *   当转出账户选“流动性资产”，转入账户选“负债（信用卡等）”，系统**强制锁定**为“转账”模式，并隐藏收支分类选项。
    *   当转出/转入账户任一为“小荷包”，自动浮出“成员标签（我/伴侣）”供勾选。
*   **分类选择器**：支出/收入模式下显示。采用两级宫格，一级不可折叠，点击展开二级列表。
*   **周期记账**：设置页配置（如每月15日公积金扣房贷），后台自动入账。

### 3.2 账户中心
*   **顶部大盘**：大字显示**净资产**（总资产-总负债），点击展开月度趋势折线图。
*   **账户分组列表**：
    *   流动性资产（余额为正）
    *   限制性资产（公积金等，余额为正）
    *   债权（借出款项，余额为正）
    *   负债（信用卡/借入款项/房贷，余额为负，红字显示）
*   **账户详情特性**：
    *   小荷包详情：顶部提供“按成员筛选”Chip。
    *   信用卡详情：区分支出（红色）与还款转账（绿色）流水。

### 3.3 统计分析页
*   **基础视图**：月度收支柱状图、一级分类支出饼图、二级排行。
*   **深度洞察卡**：
    *   **真实居住成本**：自动聚合 [支出-房贷/房租] + [支出-水电煤网] + [公积金扣款额]。
    *   **拿铁因子吞噬机**：动态聚合所有在分类设置中开启了“☕标记为拿铁因子”的二级分类支出（如零食饮料、打车出行、兴趣潮玩），并提示年度复利损失。
*   **标签聚合**：提供标签列表，点击某标签（如 #越野跑），汇总带有该标签的所有流水总额（解决跨分类算总账的痛点）。

### 3.4 我的/设置页
*   **安全锁**：APP启动/切回前台需输入 PIN 码（存本地 localStorage 加盐哈希）。
*   **数据管理**：
    *   导出备份（将 IndexedDB 序列化为 JSON 文件下载）。
    *   导入恢复（读取 JSON 文件覆盖当前数据库）。
    *   彻底销毁（清空 IndexedDB、注销 Service Worker、清空 Cache，重置为出厂状态）。
*   **分类/标签管理**：允许用户隐藏不需要的二级分类，允许新建标签，**允许用户将任意二级分类标记为“拿铁因子”**。

## 5. 数据库设计说明书 (基于 Dexie.js / IndexedDB)

### 5.1 表结构定义

```javascript
import Dexie from 'dexie';

export const db = new Dexie('MoneyBookDB');
db.version(1).stores({
  // 账户表
  accounts: '++id, groupId, name, type, icon, balance, isCommon, sort',
  // 分类表 (isLatteFactor: 是否为拿铁因子)
  categories: '++id, type, parentId, name, icon, attribute, isLatteFactor, isHidden, sort',
  // 流水表 (tags: 数组，存标签名)
  transactions: '++id, type, amount, fromAccountId, toAccountId, categoryId, tags, note, date'
});
```

### 5.2 流水表 (Transactions) 核心逻辑设计

为了实现“转账隔离”，流水表的设计必须严谨：

*   **如果是支出**：`type='expense'`, `fromAccountId=支付账户(如信用卡)`, `toAccountId=null`, `categoryId=支出分类`
*   **如果是收入**：`type='income'`, `fromAccountId=null`, `toAccountId=接收账户`, `categoryId=收入分类`
*   **如果是转账**：`type='transfer'`, `fromAccountId=转出账户`, `toAccountId=转入账户`, `categoryId=null`

### 5.3 事务一致性保障 (Dexie Transaction)

任何涉及金额变动的操作（新增、修改、删除流水），必须同步更新 `accounts` 表的 `balance`。此操作强制包裹在 `db.transaction` 中，保证流水与余额的原子性更新，防止数据错乱。

## 6. 技术架构与细节说明书

### 6.1 技术栈清单与职责

| 模块   | 选型                    | 核心职责                               |
| :----- | :---------------------- | :------------------------------------- |
| 框架   | Vue 3 (Composition API) | 构建响应式UI与业务逻辑组合             |
| 构建   | Vite                    | 极速开发服务器与生产打包               |
| UI库   | Vant 4                  | 移动端组件（重点：NumberKeyboard）     |
| 数据库 | Dexie.js                | 优雅操作 IndexedDB，提供 liveQuery     |
| 状态   | Pinia                   | 管理全局状态（分类树、当前选中账户）   |
| 图表   | ECharts (按需引入)      | 绘制统计页大盘图表                     |
| PWA    | vite-plugin-pwa         | 生成 Service Worker，配置离线缓存      |
| 工具   | @vueuse/core            | useObservable 桥接 Dexie 与 Vue 响应式 |

### 6.2 响应式数据流架构 (替代 SQL GROUP BY)

采用**“本地优先”架构的内存聚合计算**：
1.  **数据源**：Dexie `db`。
2.  **响应式桥接**：通过 `@vueuse/core` 的 `useObservable` 将 Dexie 的 `liveQuery` 转换为 Vue 的 `ref`。数据一变，UI自动更新。
3.  **计算层**：在 Vue 组件中使用 `computed`，对拉取到内存的数组进行 `reduce`/`filter` 聚合计算（如计算净资产、拿铁因子总支出）。
4.  **UI渲染**：计算结果变化，ECharts 自动重绘。

### 6.3 PWA 与离线/持久化保障

*   **manifest.json**：配置应用名、图标，`display: standalone` 隐藏浏览器UI。
*   **Service Worker**：通过 `vite-plugin-pwa` 自动生成，将所有静态资源强缓存，实现断网秒开。
*   **持久化防线**：App 首次加载时调用 `navigator.storage.persist()`，向浏览器申请免清理保护，防止磁盘空间不足时 IndexedDB 被自动清除。
*   **安全防线**：PIN 码应用锁，JSON 加密导出备份，一键彻底销毁数据功能。

### 6.4 大屏适配策略 (响应式看板)

利用 `@vueuse/core` 的 `useBreakpoints` 监听屏幕宽度：
*   **移动端 (<=768px)**：底部Tab导航，全屏键盘录入，单列图表。
*   **桌面端 (>768px)**：左侧侧边栏导航，记账改为弹窗模式，统计页开启“看板模式”（多图表并排展示），数据同源同步。
